# .planning/

Project planning and activity tracking directory for Claude Code sessions.

## Files

| File | Purpose | Updated |
|------|---------|---------|
| `activity-log.md` | Per-session action log (what changed, why, outcomes) | Every session |

## Usage

- **Start of session**: Read `activity-log.md` to understand recent context
- **End of session**: Append an entry with actions, decisions, outcomes, and next steps
- **Before major changes**: Check recent entries for related work or known issues

## Conventions

- Entries are reverse-chronological (newest first after the format section)
- Include branch name and approximate duration
- Reference specific file paths for code changes
- Note any decisions that deviate from standard patterns (and why)
